# 6エージェント自動化運用設計
## Tokyo307inc × OpenClaw完全自動化モデル

**作成日:** 2026年2月24日  
**参考:** Chris氏のWebサイト制作自動化事例  
**対象:** ホテルDX × Threes提携拡大

---

## 🎯 全体像：6エージェント連携フロー

```
① Scout（リード発掘）
    ↓ 優先リスト
② Intel（課題分析）
    ↓ 営業ブリーフ
③ Builder（デモ・提案書自動生成）
    ↓ デモ + 提案書
④ Outreach（パーソナライズ営業）
    ↓ 返信
⑤ Closer（商談準備）
    ↓ 商談
⑥ Growth（既存顧客管理・アップセル）
    ↓ 継続売上
```

**人間（八坂さん）が関わるのは「⑤商談」のみ**

## 🤖 各エージェントの役割と指示内容

### ① Scout（リード発掘エージェント）

**役割:** ホテル・宿泊施設の自動発掘とスコアリング

**指示内容:**
```
毎朝7時に起動。
以下の手順で全国のホテルを発掘し、優先リストを作成：

1. Google Places API で「ホテル」「ビジネスホテル」「カプセルホテル」を検索
   対象エリア: [東京23区、大阪市、名古屋市、福岡市、札幌市]
   
2. 各ホテルのWebサイトを取得
   - サイトなし → スコア: 100（最優先）
   - サイトあり → PageSpeed Insightsで採点
   
3. スコアリング基準:
   - Webサイトなし: 100点
   - PageSpeed 0-49: 90点（超低速）
   - PageSpeed 50-89: 70点（改善余地大）
   - 予約システムなし: +20点
   - 電話番号不明瞭: +10点
   
4. スコア順にランキング化し、上位50件をIntelに引き渡し

保存先: /workspace/leads/scout-results-YYYY-MM-DD.json
```

**使用ツール:**
- Google Places API
- PageSpeed Insights API
- Puppeteerでサイトクロール

### ② Intel（課題分析エージェント）

**役割:** ホテルサイトを診断し、営業ブリーフを作成

**指示内容:**
```
Scoutから受け取った上位50件を分析：

1. 各ホテルサイトをクロール（Puppeteer）
   チェック項目:
   - 表示速度（モバイル/デスクトップ）
   - 予約ボタンの有無・位置
   - 電話番号の視認性
   - Threesとの相性（ラウンジ/ワークスペースの有無）
   - 競合比較（周辺ホテルとの価格差）
   
2. 技術レポートではなく「営業ブリーフ」を作成
   例:
   「このビジネスホテルは駅徒歩3分の好立地だが、
    Webサイトは存在せず、Googleマップの口コミも放置。
    周辺ホテルは予約システムで直販を強化しているのに、
    このホテルはOTAに全依存し、手数料で年間500万円を損失。
    
    Threesとの相性:
    1Fにラウンジあり、平日午前の稼働率30%。
    テレワーク需要で月20万円の追加売上が見込める。」
   
3. 営業ブリーフをBuilderに引き渡し

保存先: /workspace/leads/intel-brief-YYYY-MM-DD/
```

**使用ツール:**
- Puppeteer（サイトクロール）
- Claude（ブリーフ作成）
- Google Maps API（口コミ分析）

---

### ③ Builder（デモ・提案書自動生成エージェント）

**役割:** パーソナライズされたデモサイト・提案書・動画を自動生成

**指示内容:**
```
Intelから受け取ったブリーフを基に、営業前にデモを完成させる：

1. ホテル専用Webサイトのデモ生成
   - HTML/CSSで予約システム付きサイトを自動生成
   - ホテル名、住所、写真（Google Mapsから取得）を埋め込み
   - Vercel/Netlifyに自動デプロイ
   
2. デモサイトのスクロール動画を録画
   - Puppeteer + ffmpegで30秒の画面録画
   - 「Before（現状）」と「After（デモ）」の比較動画
   
3. Threes提携提案のUGC動画を生成
   - スクリプト作成（Claude）
     例: 「平日午前のラウンジ、実はリモートワーカーの
          宝庫なんです。Threesと提携すれば、稼働率ゼロの
          時間帯が月20万円の売上に変わります」
   - 音声生成（ElevenLabs）
   - 動画生成（Nano Banana Pro or Kling）
   
4. 提案書を自動生成（Google Docs形式）
   - 課題の可視化
   - 解決策（Webサイト刷新 + Threes提携）
   - ROI試算（年間売上増加額）
   - デモサイトリンク
   
5. すべてをOutreachに引き渡し

保存先: 
- デモサイト: https://demo-hotel-[name].vercel.app
- 動画: /workspace/demos/[hotel-name]-video.mp4
- 提案書: Google Drive「AI納品/営業提案」
```

**使用ツール:**
- Claude（サイト生成、スクリプト作成）
- Puppeteer + ffmpeg（動画録画）
- ElevenLabs（音声生成）
- Nano Banana Pro（動画生成）
- Vercel/Netlify（デモサイトデプロイ）
- Google Docs API（提案書作成）

### ④ Outreach（営業エージェント）

**役割:** パーソナライズされた営業メール・SMS自動送信

**指示内容:**
```
Builderから受け取ったデモ・提案書を使って営業開始：

1. 初回メール送信（Gmail API）
   件名: 「【[ホテル名]様専用】平日午前の売上を月20万円増やす提案」
   
   本文構成:
   - 冒頭: Intelブリーフの課題を具体的に指摘
     例: 「貴ホテルのWebサイトを拝見したところ、
          モバイル表示に9秒かかり、予約ボタンが見つかりません」
   - 中盤: デモサイト・動画・提案書リンク
   - 末尾: 「15分のオンライン相談で詳細をご説明します」
   
2. SMS送信（Twilio）※日本では控えめに
   「[ホテル名]様、Tokyo307inc八坂と申します。
    先ほどメールをお送りしました。ご確認いただけますと幸いです」
   
3. 自動フォローアップ
   - 3日後: 「ご確認いただけましたでしょうか」
   - 7日後: 「他のホテル様の導入事例をご紹介します」
   - 14日後: 最終フォロー
   
4. 返信があった場合、Closerに引き渡し

送信ログ: /workspace/outreach/sent-YYYY-MM-DD.json
```

**使用ツール:**
- Gmail API（メール送信）
- Twilio（SMS送信）※日本では慎重に
- HubSpot CRM（送信履歴記録）

### ⑤ Closer（商談準備エージェント）

**役割:** 商談前のブリーフ作成（★人間が関わる唯一の場所★）

**指示内容:**
```
返信が来たら、八坂さんの商談準備を完全自動化：

1. 商談ブリーフ作成
   - ホテルの課題（Intelブリーフから）
   - 想定される反論と切り返し
     例: 「予算がない」→「初期費用ゼロのプランもご用意」
   - トークの切り口
     例: 「Threesは既に[競合ホテル名]様でも導入済みです」
   
2. Googleカレンダー連携
   - 八坂さんの空き時間を自動取得
   - Calendlyリンクを返信メールに挿入
   
3. 商談当日の朝、リマインド
   - Slackの#salesチャンネルに投稿
   - 「本日10時: [ホテル名]様との商談。ブリーフはこちら↓」

4. 商談後、結果をGrowthに引き渡し

保存先: /workspace/closer/brief-[hotel-name].md
```

**使用ツール:**
- Claude（ブリーフ作成）
- Google Calendar API
- Calendly API
- Slack API（リマインド）

### ⑥ Growth（既存顧客管理・アップセルエージェント）

**役割:** 契約後の運用監視とアップセル

**指示内容:**
```
契約成立後、自動で顧客をフォロー：

1. 月次レポート自動送信
   - Webサイトのパフォーマンス（PV、予約率）
   - Threesの利用状況（予約件数、売上）
   - 改善提案
   
2. アラート検知
   - サイトの表示速度が悪化 → 「最適化が必要です」
   - Threes予約数が減少 → 「プロモーション提案します」
   - 90日間アップセルなし → 八坂さんにアラート
   
3. アップセル提案の自動生成
   - 「eSIMホテルダイレクト」の提案
   - 「TomaRuyo掲載」の提案
   - 「Hotel Agent Pack」の提案
   
4. 契約更新リマインド
   - 契約終了30日前に自動通知

保存先: Google Drive「既存顧客管理」
```

**使用ツール:**
- Google Analytics API（サイト分析）
- HubSpot CRM（顧客管理）
- Claude（レポート・提案書作成）
- Gmail API（自動送信）

## 🔐 セキュリティ対策

### APIキー管理
- すべてのAPIキーは環境変数で管理
- ハードコード禁止

### データ保護
- 個人情報（ホテル名、連絡先）は暗号化
- Google Drive上での権限管理

### ログ記録
- すべての営業活動をHubSpot CRMに記録
- 不審な動作をmemory/security-log.mdに記録
